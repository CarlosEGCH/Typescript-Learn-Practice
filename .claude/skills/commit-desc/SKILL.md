# /commit-desc — Generate Commit Message & PR Description

Generates a ready-to-copy commit message and PR description based on the current git changes.

## Steps

1. Run `git status` to see what files are changed, staged, or untracked.
2. Run `git diff --staged` to see staged changes. If nothing is staged, run `git diff` for unstaged changes instead. If both are empty, run `git diff HEAD~1` to inspect the last commit.
3. Run `git log --oneline -5` to understand the commit style and recent history of this repo.
4. Analyze the changes and produce the output below.

## Output Format

Produce two clearly separated, copyable blocks:

---

### Commit Message

Shouldn't exceed 300 characters in the subject line


```
<type>(<scope>): <subject>
```

Use conventional commit types: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `style`.

---

### PR Description

```
## Summary
- <bullet 1>
- <bullet 2>

## Changes
- <file or area>: <what changed and why>

## Test plan
- [ ] <how to verify this works>
```

---

## Guidelines

- Write the commit message in the imperative mood ("add", "fix", "update" — not "added" or "adds").
- Keep the commit subject line under 72 characters.
- In the PR description, focus on *why* the change was made, not just *what* changed.
- If changes span multiple concerns, note that in the summary and suggest splitting into separate commits.
- Do not include boilerplate like "Generated by Claude" in the output blocks — keep them clean and directly copyable.
